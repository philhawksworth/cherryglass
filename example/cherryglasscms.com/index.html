What is it?

CherryGlass provides a simple content managment tool for your static sites. It is designed to be simple for content editors, while being kind to developers.

This site is managed with CherryGlass, and you can see that it leaves no traces behind in the HTML page source

CherryGlass separates the content management from the hosting of your site, so you can host on cost-effective and high performance static hosting providers like GitHub Pages, Amazon S3 or even Dropbox, and yet enjoy the benefits of a CMS for making easy modifications.

How does it work?

By looking for HTML5 data attribute in your markup, CherryGlass generates a content admin site for you which includes just the controls that you need.

The code for an editable element would look like this: 

<p data-cherry-id="intro" data-cherry-type="text">
 Woohoo! This will get an admin UI
</p>

Since we only need to add valid HTML attributes to your site's source, you'll still be able to view it and validate it in the same way as you did before. Nothing to mangle your rendering, even at this stage.

A common workflow

Let's looks at a common process of using CherryGlass to add a CMS to a static site.

1. Write your HTML

You'll do this better than a CMS engine. We'll let you engineer your front-end code the way you like it. Whatever your usual process is, do that.

2. Create content 'cherries'

Decide which page elements you need to be managed by an editor, and add HTML5 data attributes to those. CherryGlass will cherry-pick those for administration.

3. Generate a CMS admin site

This where CherryGlass kicks in. Feed it your HTML and it will poop out a CMS admin site for you. The site runs on NodeJS and you can run it locally or on a development server.

4. Edit the content

Use the simple form-based admin site to edit the elements you specified.

5. Build and Deploy

At the push of a button, CherryGlass will combine yout HTML with the values entered in the admin site. It will clean up all of the data attributes it used and output your HTML. Fresh and free from any sign of the CMS which generated it.

Deploy your site to any static server. Done.

Hosting possibilities

Your site output is simple to host. It is a collection of static files which means it will fly if you host it somewhere like GitHub Pages, Amazon S3 or even Dropbox.

But what about hosting the admin site? The choice is yours, but some scenarios are:

  • Host it on a development or staging server for your content editting team to use.
  • Push your HTML to us and let us generate and host your admin site for you. Yep, that's CMS As A Service. We can generate your site from your HTML and the content you manage on the site, and then push the output wherever you like.

Documentation

You can use CherryGlass without installing anything at all. Just take one minute to register and then start pushing your HTML to us.

It's better to test your CMS locally first though. Luckily, installing CherryGlass and giving it a spin is simple.

Installation

Ensure that NodeJS is installed before we begin. Then clone the project into your project directory: 

$ git clone https://github.com/philhawksworth/cherryglass.git

Install the dependancies with NPM (which is installed with NodeJS) 

$ cd cherryglass
$ npm install
$ cd ..

Run CherryGlass with NodeJS 

$ node cherryglass

Now you can point your browser at http://localhost:3000/cms and start managing content.

Syntax reference

CherryGlass is intentionally simple, so there is not a lot of syntax to learn. Just a few data attributes.

AttributeDescription
data-cherry-idA unique identifier on the page. (required)
data-cherry-typeThe type of element to manage. See below. (required)
data-cherry-labelOptionally add a friendly label for this element when it appears in forms of the admin site. Otherwise you'll see it labeled with its ID.
data-cherry-helpOptionally add some description or guidance in the admin site for this element.

The data-cherry-type attribute supports the following content types:

TypeDescription
textA simple text field. Admin site shows a text input field
blobA larger text field. Admin site shows a textarea input field
markdownText entered in markdown is interpreted and injected into the site as HTML. Admin site shows a textarea input field
