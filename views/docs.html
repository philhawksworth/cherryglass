Documentation

GlassCherry CMS is a light-weight tool which adds a simple CMS to your static website.

You've installed it (thanks!) as a tool that runs in Node. Now you'll be able to use it to create a CMS admin site for your project, based on your existing HTML code.

Cherry syntax

HTML elements in your page source can be made content-manageable by marking them as content cherries. GlassCherry will cherry-pick just these elements to construct a set of admin pages for you.

Marking an element as a cherry is done by adding the data attribute of data-cherry to it. This validates with the HTML5 spec and does not impact the rendering of your pages when viewed as normal before conten management.

The format for the data-cherry attributes are: 

<h1 data-cherry-id="page-title" data-cherry-type="text">Placeholder text</h1>

id and type are the only mandatory attributes in cherries.

Cherry attributes

data-cherry-id="any string"
(required) A unique identifier on the page for this content. Displayed as a label for the field in the admin pages if no label is explicitly provided.
data-cherry-type="[text | blob | markdown | img | collection]"
(required) Defining the type of content to deliver through the admin UI. Details below
data-cherry-label="any string"
(optional) A label for displaying with the field in the admin pages.
data-cherry-help="any string"
(optional) Descriptive content displayed in the admin pages to give guidance for the purpose or usage of this field.

Cherry types

data-cherry-type="text"
Provides a simple text element for management. Good for short string substitutions.
data-cherry-type="blob"
Provides a larger text element for management. Shows as a textarea in the admin pages. Good for longer sections of text but supports no formatting other than paragraphs.
data-cherry-type="markdown"
Provides a larger text element for management. Shows as a textarea in the admin pages and supports markdown for simple formatting.
data-cherry-type="img"
Provides management of the source of an image asset. The admin pages will show an image upload form.
data-cherry-type="collection"
Provides a nested, repeatable set of elements. The admin pages give the option to add new elements within that collection
